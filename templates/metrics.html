Methodology

Engineered features, categorical encoding, validation, and real-time fusion delivery.

01

Unified input schema

One common transaction form captures amount, location, merchant category, card type, behavior velocity, distance, and card presence signals.

02

Feature engineering

Derived fields such as `is_night`, `high_amount`, `location_change`, and `device_trust_score` sharpen the behavioral model before classification.

03

Independent training

The Indian model learns transaction behavior from the regional dataset, while the global model learns card-level anomalies from the European fraud dataset.

04

Fusion decision

Flask serves a single prediction endpoint that combines both probabilities and returns one fraud score, one risk level, and one explanation set.
