Methodology

Simple enough to explain, robust enough to demonstrate live.

01

Preprocessing

Numeric behavior features are standardized for Logistic Regression, while binary indicators remain easy to interpret.

02

Imbalance handling

Fraud is intentionally rare in the dataset, and the pipeline uses class balancing plus SMOTE to stay sensitive to suspicious cases.

03

Evaluation and tuning

Precision, recall, F1-score, ROC-AUC, average precision, and cross-validation are used before selecting a tuned threshold.

04

Deployment

The best model is persisted and served through a Flask API that the UI calls in real time during the demo.
