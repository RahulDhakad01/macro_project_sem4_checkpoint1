About Sentinel Fraud Shield

🎓

Project overview

Real-time fraud decision support

Sentinel Fraud Shield is an end-to-end credit card fraud detection system that combines two fraud datasets, two machine learning models, and one fused decision layer to screen transactions in real time with explainable output.

🤖

Prediction engine

Unified input, dual-model scoring

One unified transaction form captures amount, hour, location, merchant category, card type, transaction velocity, foreign usage, card presence, and distance signals before sending the same profile into both fraud models.

🩺

Missing value handling

Stable preprocessing pipeline

Missing numeric values are handled with median imputation, categorical values use most-frequent imputation, and encoding, scaling, and feature preparation are embedded directly inside the training pipelines for reliable prediction behavior.

⚖️

Why this helps

Faster screening, clearer actions

The system converts multiple fraud indicators into one fraud score, one risk level, and dynamic reasons such as unusual hour, high amount, new location, foreign usage, or high transaction frequency, making the decision easier to understand.

📊

Dataset

Two datasets, two learning styles

The Indian dataset contributes behavioral features such as location, card type, purchase category, customer age, and fraud labels across {{ dashboard.datasets.indian_rows }} rows, while the European Kaggle dataset contributes Time, Amount, and V1-V28 anonymized card-pattern features across {{ dashboard.datasets.european_rows }} rows.

🛠️

Methodology

Algorithms and fusion workflow

The Indian behavioral dataset is trained using Random Forest, the European dataset is trained using Logistic Regression, and the final risk score is fused at {{ dashboard.fusion_weights.indian }} Indian plus {{ dashboard.fusion_weights.global }} global before dynamic reason generation is returned to the user.
